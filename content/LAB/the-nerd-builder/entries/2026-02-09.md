## 11:24 — Devlog : l’illusione del “funziona”

Ora: abbiamo una web app da personalizzare.

Al codice (come sempre) ci pensa ChatGPT.
Alle idee e alle imprecazioni ci penso io.

Mettiamo giù l’idea iniziale:

Un file indice con tutti gli articoli: titoli, sezioni, tag, date… e l’indirizzo degli articoli veri e propri.
JSON, perché sì: voglio ordine.

Una cartella per ogni progetto.
Dentro ci mettiamo il primo .md con tutto quello che c’è da sapere nella fase iniziale.

Un file .md datato per ogni aggiornamento.
Così carichiamo tutto e otteniamo il nostro bell’articolo, impacchettato come si deve.

Perfetto.
Lanciamo l’idea, incolliamo il codice… e poi:


```bash
npm run dev
```

Vediamo che succede.

Funziona.
Decisamente bene. Ordinato. E stranamente anche le path funzionano (che non è mai scontato).

Facciamo un paio di ritocchi… poi riscriviamo tutto…
e all’improvviso smette di funzionare.

Ebbene sì: ChatGPT è bravo a scrivere codice tanto quanto a dimenticarlo.

Da lì parte l’odissea:
merge di file vecchi e nuovi, correzioni su correzioni, chat intasate, file troncati…
e quella sensazione costante che il progetto stia sempre a un passo dal diventare un puzzle senza immagine sulla scatola.
